# Users Context

Starter context matching the backend `users` context.

Add user-specific frontend code here only when needed:

- `domain`: frontend user entities, models, and lightweight value objects.
- `application`: user commands, queries, events, handlers, services, and facades.
- `infrastructure`: user API clients, request/response DTOs, mappers, and repositories.
- `presentation`: user pages and Atomic Design components based on Flowbite.

Do not copy backend user business rules into the frontend.
