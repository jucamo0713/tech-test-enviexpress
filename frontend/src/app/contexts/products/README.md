# Products Context

Starter context matching the backend `products` context.

Add product-specific frontend code here only when needed:

- `domain`: frontend product entities, models, and lightweight value objects.
- `application`: product commands, queries, events, handlers, services, and facades.
- `infrastructure`: product API clients, request/response DTOs, mappers, and repositories.
- `presentation`: product pages and Atomic Design components based on Flowbite.

Do not copy backend product business rules into the frontend.
