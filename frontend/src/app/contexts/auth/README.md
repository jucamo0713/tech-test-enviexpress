# Auth Context

Starter context matching the backend `auth` context.

Add authentication-specific frontend code here only when needed:

- `domain`: frontend auth entities, models, and lightweight value objects.
- `application`: auth commands, queries, events, handlers, services, and facades.
- `infrastructure`: auth API clients, request/response DTOs, mappers, and repositories.
- `presentation`: auth pages and Atomic Design components based on Flowbite.

Do not copy backend authentication business rules into the frontend.
